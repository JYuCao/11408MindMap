## 计算机组成原理（计组）

### 存储系统 / Cache / 总线
* 主存与 CPU 的连接（字扩展、位扩展）：字扩展数据线、位扩展地址线 
* Cache 的命中率分析（直接映射、组相联等） 🌟🌟
* 多体交叉存储，流水线计算（练题）
* CPU、总线主频、Cache 等执行时间综合题（综合性能计算）
* IO总线
  * 数据线传输信息：命令字、状态字、中断类型号
  * 地址线传输信息：物理地址、虚拟地址
  * 控制线传输信息：读/写信号、时钟信号
* DRAM
  * 地址复用技术，地址线减少一半
  * 刷新：行刷新、周期刷新
* 指令Cache与数据Cache分离的优点：减少结构冒险
* 微指令下地址位数的计算
* 异步、半同步、同步总线的区别
  * 异步总线：无时钟信号，靠握手信号协调数据传输
  * 半同步总线：握手信号与时钟信号结合使用，发送端提供时钟
  * 同步总线：系统统一时钟信号，数据传输完全依赖时钟

### CPU / 指令流水线
* 指令流水线，以及 TLB 缺失和 Cache 缺失 
	* IF：取指，访问 Cache/MEM
	* ID：译码，涉及寄存器取数
	* EX：执行，不涉及访问存储器
	* MEM：访存，写/读 Cache 或 MEM
	* RB：写回，写寄存器
* 指令流水线的冒险与处理方法 
	* 结构冒险：停顿（stall）或增加硬件单元
	* 数据冒险（写后读 RAW）：延迟（插入 nop）或转发（旁路）EX->EX
	* load-use：通常需要延迟 1 周期，或使用 MEM->EX 转发
	* 控制冒险：分支预测或插入延迟槽/停顿
* 中断处理会保存而子程序调用不保存的寄存器：PSW（思考：为什么？）

### 算术与数表示（真题要点）
* 无符号数、有符号数、浮点数的溢出判断 
* 乘法溢出判定 ：
	* 无符号数乘法溢出：高 n 位不全为 0
	* 有符号数乘法溢出：高 n+1 位不全为 0 或 1
* 除法溢出、浮点数相关运算（练习题）
* 时钟周期、总线周期和总线带宽的关系 ：总线时钟频率由更高的频率分频得出；总线周期频率需根据总线时钟得到
* 浮点数与IEEE 754标准🌟🌟
  * 32位浮点数：
    * $$ \text{value} = (-1)^S \times 1.F \times 2^{E-127} $$
    * 阶码范围：$[1, 254]$，其中 0 表示非规格化数，255 表示无穷大或 NaN
    * 尾数全一可以表示为 $ 2 - 2^{23} $
  * 浮点数结构：
    * 128位浮点数：1 位符号位，15 位阶码，112 位尾数
    * 64位浮点数：1 位符号位，11 位阶码，52 位尾数
    * 32位浮点数：1 位符号位，8 位阶码，23 位尾数
    * 16位浮点数：1 位符号位，5 位阶码，10 位尾数
* 对阶、左规、右规、舍入
  * 对阶：将阶码小的数的阶码调整为与大的数相同
  * 左规：将尾数左移，直到最高位为 1，同时阶码减小
  * 右规：将尾数右移，直到最高位为 1，同时阶码增大
  * 舍入：根据需要对尾数进行舍入，常用的方法有截断、向零舍入、向正无穷舍入、向负无穷舍入、四舍五入
* 纠错码/海明码
  * 海明码：增加 k 位校验位，能检测并纠正单比特错误，检测双比特错误
  * 海明码校验位位置：2 的幂次方位置
  * 海明码校验位计算：覆盖位置上所有二进制表示中对应位为 1 的数据位进行异或
  * 纠错位数：满足 $2^k \geq m + k + 1$，其中 m 为数据位数，k 为校验位数

## 操作系统（OS）

### 进程与并发
* 进程基本概念、进程状态转换
* 进程间同步／死锁与信号量（练题）
* 皮特森算法 ：用于两个进程互斥，利用 flag[i] 和 turn 实现互斥、无饥饿、有界等待
* 创建新进程的主要原因：用户登录、高级调度、系统/应用请求
* 进程进入临界区时，不一定能进行调度（内核态临界区不能）

### 内存管理
* 请求分页管理 ：快表（TLB）缺失后的更新通常由硬件自动完成，并行于后续访存；快表缺失情况下理论上只需一次快表访问（注意题目语境）
* 请求分页系统：
  * 页面分配策略
    * 固定分配：运行前分配一组固定页框，后无法更改
    * 可变分配：运行时根据需要动态分配页框
  * 页面置换策略
    * 全局置换：所有进程共享所有空闲页框
    * 局部置换：每个进程只能在其分配的页框中进行置换

### 文件与磁盘
* 文件控制块（FCB）与文件分配表（FAT） ：FCB 为打开文件表提供信息，FAT 位于分区开头记录簇链
* 磁盘读取时间 = 寻道时间 + 旋转延迟 + 传输时间；平均旋转延迟 = 单圈时间 / 2
* SCAN、C-SCAN、LOOK、C-LOOK 磁盘寻址策略及区别 
* 提高RAID可靠性的方法 ：奇偶校验、磁盘镜像
* 与单个文件长度相关的因素有：间接地址索引级数、块大小、地址项个数；无关的有：索引节点个数（决定文件个数）
* 改进型Clock算法优先替换脏页而非访问页：增加可控性，在空闲时主动将脏页优先写回
* 不会导致磁盘臂黏着的算法：FCFS（先来先服务）
* 索引节点相关题目

### 设备 / 驱动
* 设备驱动程序：运行在内核态、允许可重入、与硬件密切相关（部分用汇编实现），负责在进程 I/O 请求与设备控制器之间做翻译 
* 定时、中断、DMA综合题（练习题）

### 中断处理
* 单级中断过程🌟🌟
  * 硬件关中断、保存断点与现场、识别中断源；中断处理程序处理事件并恢复现场
* 中断方式的数据传输通过**软件**完成，DMA方式通过**硬件**完成

## 计算机网络（CN）

### 物理层
* 香农定理与时延-带宽积 
* QAM 调制 ：传输速率 R = B log_2 m n；最大（奈奎斯特） R = 2B log_2 m n
* 波特率相关知识 

### 传输层
* 分组重排与分片重组 （注意分片重组为网络层任务，但与传输层序号和重传逻辑相关）
* TCP 计时器 ：重传计时器（RTO）、持续计时器、保活计时器、TIME-WAIT 计时器（通常为最长报文生存期的两倍）
* TCP 的建立与释放（三次握手、四次挥手）
* 拥塞控制机制与计算 （练题重点）

### 网络层 / 协议 
* TCP 的 seq 和 ACK 编号计算
  * seq：当前段数据的第一个字节编号
  * ACK：期望收到的下一个字节编号
* ICMP 报文类型及意义 ：目的不可达、源点抑制、时间超过、参数问题、重定向
* ICMP 协议基于 IP 协议
* 连续ARQ信道利用率的计算🌟🌟🌟
  * $U = \frac{n \times L / R}{RTT + L / R} $
### 数据链路与以太网
* 以太网有效数据传输率计算（去掉前导码等开销）
* 快速以太网最小有效载荷： 46B（总64B），小于则填充至46B
* 直通交换与存储转发交换的区别
  * 直通交换只检查目的MAC地址：6B/带宽
  * 存储转发交换检查整个帧：帧长/带宽

### 应用层
* HTTP 请求
  * connection：close（非持续连接）、keep-alive（持续连接）
  * HTTP/1.0 不支持持续连接，HTTP/1.1 支持长连接，默认持续连接
  * 持续连接下，请求n个对象需要n+1个RTT
